# استكشاف الأخطاء وحلها - Troubleshooting Guide

## 🔧 **مشاكل البناء (Build Issues)**

### **1. Gradle Sync Failed**

#### **المشكلة:**
```
Could not resolve all dependencies
```

#### **الحلول:**
```bash
# 1. تنظيف المشروع
./gradlew clean

# 2. إعادة بناء
./gradlew build --refresh-dependencies

# 3. في Android Studio
File > Invalidate Caches > Invalidate and Restart
```

---

### **2. SDK Not Found**

#### **المشكلة:**
```
SDK location not found
```

#### **الحل:**
1. افتح `File > Project Structure`
2. تحقق من `SDK Location`
3. أو أنشئ ملف `local.properties`:
```properties
sdk.dir=/path/to/android/sdk
```

---

### **3. Out of Memory**

#### **المشكلة:**
```
OutOfMemoryError: Java heap space
```

#### **الحل في `gradle.properties`:**
```properties
org.gradle.jvmargs=-Xmx4096m -XX:MaxPermSize=1024m
org.gradle.daemon=true
org.gradle.parallel=true
org.gradle.caching=true
```

---

### **4. Dependency Conflicts**

#### **المشكلة:**
```
Duplicate class found
```

#### **الحل:**
```kotlin
// في app/build.gradle.kts
configurations.all {
    exclude(group = "com.google.guava", module = "listenablefuture")
}
```

---

## 📱 **مشاكل وقت التشغيل (Runtime Issues)**

### **1. التطبيق يتوقف فجأة (Crash)**

#### **الأعراض:**
- التطبيق يغلق مباشرة بعد الفتح
- رسالة "App has stopped"

#### **الأسباب والحلول:**

**أ. نقص الصلاحيات:**
```kotlin
// تحقق من الصلاحيات
if (!PermissionUtils.hasStoragePermission(context)) {
    // اطلب الصلاحيات
    PermissionUtils.requestAllPermissions(activity)
}
```

**ب. مشكلة في قاعدة البيانات:**
```bash
# امسح بيانات التطبيق
Settings > Apps > Pro Download Manager > Clear Data
```

**ج. تحقق من Logcat:**
```bash
adb logcat | grep "AndroidRuntime"
```

---

### **2. التنزيل لا يبدأ**

#### **الأعراض:**
- الضغط على "إضافة" لا يفعل شيء
- التنزيل يبقى في "في الانتظار"

#### **التشخيص:**

**1. تحقق من الاتصال:**
```kotlin
NetworkUtils.isNetworkAvailable(context)
// يجب أن يرجع true
```

**2. تحقق من الرابط:**
```bash
# اختبر الرابط
curl -I "URL"
# يجب أن يرجع HTTP 200
```

**3. تحقق من المساحة:**
```kotlin
val freeSpace = File(downloadPath).freeSpace
// يجب أن يكون كافياً
```

**4. تحقق من عدد التنزيلات النشطة:**
```kotlin
// الحد الأقصى 3
val activeCount = downloadEngine.getActiveDownloadCount()
```

---

### **3. سرعة التنزيل بطيئة جداً**

#### **الأسباب المحتملة:**

**أ. عدد الأجزاء قليل:**
```
الحل: زد عدد الأجزاء إلى 8-16
```

**ب. الخادم لا يدعم Range:**
```
الحل: سيستخدم التطبيق جزء واحد تلقائياً
```

**ج. حد السرعة مفعّل:**
```
الحل: افتح الإعدادات > حد السرعة > غير محدود
```

**د. سرعة الإنترنت الفعلية:**
```bash
# اختبر سرعة الإنترنت
speedtest-cli
```

---

### **4. التنزيل يفشل باستمرار**

#### **رسائل الأخطاء الشائعة:**

**أ. "HTTP 403 Forbidden"**
```
السبب: الخادم يرفض الوصول
الحل: 
- تحقق من صحة الرابط
- بعض المواقع تحتاج تسجيل دخول
- جرب رابط آخر
```

**ب. "HTTP 404 Not Found"**
```
السبب: الملف غير موجود
الحل:
- تحقق من الرابط
- قد يكون الملف محذوف
```

**ج. "No space left on device"**
```
السبب: لا توجد مساحة كافية
الحل:
- احذف ملفات غير ضرورية
- انقل الملفات إلى SD Card
```

**د. "Connection timeout"**
```
السبب: الخادم لا يستجيب
الحل:
- تحقق من الاتصال
- جرب لاحقاً
- قلل عدد الأجزاء
```

---

### **5. الاستئناف لا يعمل**

#### **الأسباب:**

**أ. الخادم لا يدعم Resume:**
```
التحقق:
curl -I -H "Range: bytes=0-100" "URL"
# ابحث عن: Accept-Ranges: bytes
```

**ب. تغير الملف على الخادم:**
```
الحل: احذف التنزيل وابدأ من جديد
```

**ج. ملف التنزيل المحلي تالف:**
```
الحل: 
1. احذف التنزيل (مع الملف)
2. أعد إضافة التنزيل
```

---

### **6. الإشعارات لا تظهر**

#### **التحقق:**

**1. صلاحيات الإشعارات (Android 13+):**
```kotlin
PermissionUtils.hasNotificationPermission(context)
```

**2. إعدادات النظام:**
```
Settings > Apps > Pro Download Manager > Notifications
تأكد أنها مفعّلة
```

**3. وضع عدم الإزعاج:**
```
تحقق من أن الهاتف ليس في وضع "Do Not Disturb"
```

---

## 🐛 **مشاكل الأداء (Performance Issues)**

### **1. التطبيق يستهلك بطارية كثيرة**

#### **الحلول:**

**أ. قلل عدد التنزيلات المتزامنة:**
```
الإعدادات > الحد الأقصى للتنزيلات المتزامنة: 1-2
```

**ب. قلل عدد الأجزاء:**
```
عند إضافة تنزيل > عدد الأجزاء: 2-4
```

**ج. فعّل WiFi فقط:**
```
عند إضافة تنزيل > ✓ WiFi فقط
```

---

### **2. التطبيق يبطئ الهاتف**

#### **الحلول:**

**أ. قلل استهلاك الموارد:**
```
الإعدادات > تحسينات الأداء:
- قلل عدد التنزيلات المتزامنة
- قلل عدد الأجزاء
- فعّل "توفير الطاقة"
```

**ب. أغلق التطبيقات الأخرى:**
```
أغلق التطبيقات غير الضرورية
```

---

### **3. التطبيق يستهلك بيانات كثيرة**

#### **الحلول:**

**أ. استخدم WiFi فقط:**
```
✓ WiFi فقط (في كل تنزيل)
```

**ب. حدد التنزيلات:**
```
نزّل فقط الملفات الضرورية
```

---

## 📊 **حالات الاستخدام الخاصة**

### **1. تنزيل ملفات كبيرة جداً (>5 GB)**

#### **التوصيات:**
```
✓ استخدم 16 جزء
✓ WiFi فقط
✓ أولوية عالية
✓ أغلق التطبيقات الأخرى
✓ تأكد من مساحة كافية (ضعف حجم الملف)
```

---

### **2. تنزيل على Mobile Data**

#### **التحذيرات:**
```
⚠️ قد يستهلك رصيدك
⚠️ قد يكون أبطأ
⚠️ قد ينقطع الاتصال
```

#### **التوصيات:**
```
✓ قلل عدد الأجزاء (2-4)
✓ راقب استهلاك البيانات
✓ استخدم حد السرعة إذا لزم
```

---

### **3. تنزيل من مواقع تحتاج تسجيل دخول**

#### **المشكلة:**
```
التطبيق لا يدعم Cookies/Sessions حالياً
```

#### **الحل البديل:**
```
1. استخدم متصفح للحصول على رابط مباشر
2. أو استخدم IDM/ADM للمواقع التي تحتاج تسجيل دخول
```

## ✅ **نصائح عامة لأداء أفضل**

1. ✅ **حدّث التطبيق دائماً** لآخر إصدار
2. ✅ **امنح جميع الصلاحيات** المطلوبة
3. ✅ **استخدم WiFi** للتنزيلات الكبيرة
4. ✅ **اترك مساحة فارغة** كافية (20% على الأقل)
5. ✅ **أعد تشغيل الهاتف** بين الحين والآخر
6. ✅ **نظّف ذاكرة التخزين** المؤقت بانتظام
7. ✅ **لا تنزّل ملفات كثيرة** دفعة واحدة
8. ✅ **راقب درجة حرارة الجهاز** أثناء التنزيل
